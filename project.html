Designing for Accessibility

Enhancing usability for screen reader users through inclusive design

Overview

This project focused on improving the accessibility and usability of an educational platform for users who rely on screen readers. Through user interviews, heuristic evaluations, and iterative usability testing, we uncovered major pain points and introduced design solutions that significantly improved user experience.

My Role

  • Led user interviews with visually impaired participants
  • Conducted accessibility audits following WCAG 2.1 standards
  • Collaborated with developers to ensure ARIA implementation accuracy
  • Designed and tested inclusive prototypes using Figma

Process

I began by conducting contextual inquiries and accessibility walkthroughs using NVDA and VoiceOver. After identifying key usability barriers, I redesigned navigation patterns and improved heading structures. The new design was tested with 5 users, leading to a 45% improvement in task completion time.

Outcome

The redesign not only improved accessibility but also boosted overall engagement metrics. The project was later adopted as part of the platform’s official accessibility improvement roadmap.

“The new layout makes everything easier to find — I feel like I can actually use the site without frustration now.”
— Participant feedback
